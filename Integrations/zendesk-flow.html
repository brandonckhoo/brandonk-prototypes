Flowchart illustrating how Retell AI agents connect to Zendesk to look up customer history, create or update tickets,
add comments, and log call outcomes back into the system of record.
How it works
1
Voice conversation starts. A customer calls in and speaks with the Retell AI voice agent. Retell handles the conversation in real time using the agent's configured prompt and tools.
2
Lookup user and existing tickets. At the start of the call, Retell queries Zendesk using the caller's phone number or email to find the existing user record and any open tickets. This gives the agent full context on who is calling and what issues they already have.
3
Zendesk returns customer history. Zendesk sends back the user profile, ticket history, tags, and any custom field data. The agent now has context to greet the customer by name, reference prior issues, and avoid asking repeat questions.
4
AI agent handles the call. The agent uses the retrieved context to resolve the issue or gather the information needed for follow up. If the AI can resolve the issue, it does. If not, it collects the details required to create or update a ticket.
5
Create or update ticket. During or after the call, the agent writes back to Zendesk, creating a new ticket or updating an existing one with conversation summary, tags, priority, assignee, and custom field values. Multiple ticket forms and custom fields are supported.
6
Zendesk confirms. Zendesk returns the ticket ID and confirmation. Any Zendesk triggers, automations, or SLA rules fire based on the customer's existing configuration.
7
Agent confirms to caller. The agent tells the customer their ticket has been created or updated, giving them the ticket number and expected follow up. The customer leaves the call with a clear next step.
8
Log call outcome. After the call ends, Retell logs the full call summary, duration, recording link, and outcome back to Zendesk as a voice activity or ticket event. Support teams now have a complete interaction history inside Zendesk.
