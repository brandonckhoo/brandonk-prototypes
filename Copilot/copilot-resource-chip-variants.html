Copilot resource chip

One chip equals one whole resource. The chip is auto attached from the page the user has open when the panel opens, sits at the top of the composer, and carries the resource name plus its qualifier (time range, timestamp, or time window). Landing chips are queries (the page's filter state, resolved at ask time); detail chips are references to one entity. Removing the chip detaches the resource from context. The + button re-attaches it.

Anatomy

Call History · Jun 1 – Jun 9

Remove × — hover only, detaches the resource from context

Behavior

  • Auto attach — chip appears when the panel opens, scoped to the current page
  • Static — no intent classification, no partial fragments
  • It is a query — landing chips carry time range + all filters + sort, resolved at ask time. Full filtered set, not the rows on the current page
  • Label rule — time range always visible; other filters collapse to +N, full list on hover
  • Chip follows the page — changing the date range or filters updates it live
  • Empty result — 0 matches keeps the chip valid; Copilot reports no matching calls and offers to widen
  • Navigation swaps it — moving landing → detail replaces the chip, never stacks
  • Truncation — name truncates with ellipsis, qualifier never wraps
