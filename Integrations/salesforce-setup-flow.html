About
i Authorizing once gives every agent in this workspace access to your Salesforce data. When a call arrives, Retell looks up the caller's Contact record in real time so the agent knows who is calling before the first word is spoken. Retell also keeps Contact records in sync, pulling updates from Salesforce so the agent always has current information. When Memory is enabled, Retell persists what the agent learned from each call back to the Contact record so future calls can pick up where the last one left off.

You can also add pre-call functions on a per-agent basis to fetch Leads, Opportunities, Cases, and Accounts in real time, and post-call functions to log tasks, update records, and create new leads. The OAuth token is stored securely and never shown again after this step.
Shown in the integrations list and in the agent editor.
OAuth 2.0
Permissions Retell will request
Read and write Contact records
Read Lead, Opportunity, Account, and Case records
Create Tasks and Activity records
Access basic org information

Integrations
i
Salesforce is connected
Your Contacts are now syncing. Retell gives you three ways to use this connection.
1
Contacts and Memory are active. Contact fields sync every 5 minutes and are available as {{contact.*}} variables across every agent with no per-agent setup. Memory captures what the agent learns on each call and writes it back to the Contact record automatically.
2
Add pre-call functions for other records. Fetch Leads, Opportunities, Cases, and Accounts right before a call connects using this same Salesforce connection. No second OAuth needed.
3
Add post-call functions to write outcomes. Log tasks, update records, and create leads after every call automatically. Configure in the Functions panel of any agent.
