Flowchart illustrating how Retell AI agents sync bidirectionally with HubSpot to look up contacts,
update custom properties, manage deals, and log call activities during and after voice conversations.
How it works
0
Background sync every 30 min (runs continuously). In Phase 1, Retell does not query HubSpot on every inbound call. Instead, contact data syncs from HubSpot into Retell on a 30 minute interval in the background. The rationale: (1) querying HubSpot on every call adds API latency that hurts the caller experience, (2) every inbound call would generate extra API calls, and enterprise customers with high call volume would quickly hit HubSpot API rate limits (most tiers cap at 100 calls per 10 seconds), (3) contact data does not change frequently enough to justify the cost of per call fetches, and (4) a scheduled sync keeps the architecture simpler for Phase 1 launch. Real time per call lookups are a Phase 2 upgrade for workflows that need the freshest data.
1
Inbound call arrives. A customer calls into Retell. The agent begins the conversation with the caller's phone number captured from the incoming call metadata.
2
Read cached contact data. Retell looks up the caller in its own local contact store, which was synced from HubSpot within the last 30 minutes. The lookup is instant, no external API call required. If the caller is known, the agent has the caller's name, email, custom properties, and recent deal context from the last sync. If the caller is unknown (no match on phone number), the agent treats it as a first time contact and will create a new record post call.
3
AI agent handles call with context. For known callers, the agent greets them by name, references past interactions, and uses custom properties specific to the customer's workflow. For unknown callers, the agent uses a generic greeting, asks for name, email, and purpose naturally during conversation, and captures the data needed to create a new contact. Either way, the experience feels personalized.
4
Agent confirms next steps to caller. The agent tells the customer what will happen next, then ends the call. All the new information collected during the call is held in memory but not yet written to HubSpot. This keeps the conversation fast with zero API latency mid call.
5
Post call batch write to HubSpot. After the call ends, Retell writes everything back to HubSpot in a single batched call. For known callers: updates contact properties, custom properties, deal creation or stage change, and logs call summary, duration, recording link, outcome, and activity attached to the contact and deal. For unknown callers: creates a new contact with the data collected during the call (name, email, phone, company if captured), marked with Source "Retell AI inbound call", optionally creates a deal, and logs the first call activity attached to that contact. Batching keeps calls fast, reduces API usage, and lets HubSpot workflows fire once at the end rather than multiple times mid conversation.
→
Phase 2 upgrade: real time lookups. A future phase will add the option to query HubSpot directly on inbound call for the freshest data. This matters for workflows where a contact was created or updated in the last 30 minutes, for example a lead that just filled out a form and called in immediately after. Phase 1 scope uses the 30 minute sync to keep complexity and latency low at launch.
