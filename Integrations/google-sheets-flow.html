Proposed flow diagram (strawman for engineering review).
Single Google OAuth grants access to Sheets, Docs, and Drive. Retell syncs selected sources into its knowledge base on a scheduled interval,
then uses semantic search during live calls to retrieve Workspace content without per call API lookups.
How it works
0
Single Google OAuth connection. The customer connects their Google account once. That single OAuth grants Retell access to Sheets, Docs, and Drive in the same flow. Customers pick which spreadsheets, docs, folders, or files to sync. Org level connection ownership.
1
Scheduled sync every 30 min (configurable). Retell syncs selected Workspace sources into its knowledge base on a scheduled interval (15, 30, or 60 min). Scheduled sync avoids API latency during live calls, stays within Google API rate limits, and reuses Retell's existing document KB infrastructure.
2
Chunk and embed content into KB. Retell processes each source type appropriately: Sheets chunk by row, Docs chunk by paragraph or heading, Drive files (PDFs, Word docs) chunk by page or section. All content is embedded into the same KB index so the agent can retrieve across all three source types.
3
Customer asks a question. During the call, the customer asks something that requires Workspace content, for example "how much does SKU 1234 cost" (Sheets), "what's our refund policy" (Docs SOP), or "is the service manual available" (Drive PDF).
4
Semantic search across all sources. Retell runs a semantic search across the entire KB and returns the most relevant chunk regardless of source type. This is a local lookup inside Retell's KB, no external Google API call, no added latency.
5
Agent answers with Workspace content. The agent uses the retrieved chunk to respond naturally, citing the row, paragraph, or section that best answers the question. The caller experiences a single seamless answer without knowing which source it came from.
6
Manual refresh option. If a customer updates a Workspace source and wants the change reflected immediately, the integration settings page offers a "Refresh now" button that triggers an ad hoc sync for that source. Escape hatch for urgent updates without per call API overhead.
→
Phase 2+ consideration: live query mode. A future phase could add live Sheets API queries for customers whose data changes in real time (live inventory, dynamic pricing). Deferred because no current customer has validated this need; the 30 min sync covers the validated use cases in Phase 1.
