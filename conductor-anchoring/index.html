Conductor · Feature proposal

Node Anchoring

Conductor knows where everything it touches lives. A node, a setting, a variable, an integration. Node Anchoring makes each one clickable, so instead of telling you where to look, it takes you there and highlights it.

The problem

Conductor names what to change, but not where. Setting names, code, and placeholders like cal_api_key sit in a grey wall of text. Nothing stands out and nothing is clickable. You read the names, then go digging through the builder to find each one yourself.
