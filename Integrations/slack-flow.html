Flowchart illustrating how Retell AI agents send notifications, call summaries, and escalations
to Slack channels during and after live conversations.
How it works
1
Voice conversation starts. A customer calls in and speaks with the Retell AI voice agent. The conversation proceeds normally using the agent's configured prompt and tools.
2
AI agent handles the call. The agent conducts the conversation, resolves the customer's request, and collects relevant data such as caller intent, outcome, and next steps.
3
Trigger event detected. During the call, the agent identifies a moment that needs a team notification, such as a high value lead, an escalation request, a VIP caller, or a specific workflow trigger the team has configured.
4
Send message to Slack. Retell posts a message to the configured Slack channel using the Slack Web API. Messages can include caller details, intent, collected data, and a link back to the Retell call record. Channel mapping is set up during integration configuration.
5
Slack confirms delivery. Slack returns the message ID and timestamp confirming the message was delivered to the channel. The team is now notified in real time, inside the tool they already use.
6
Call ends. The agent wraps up the conversation with the customer.
7
Post call summary to channel. After the call ends, Retell posts a complete call summary to Slack including duration, outcome, key data points, and a link to the call recording or transcript. Teams can follow up without switching tools.
