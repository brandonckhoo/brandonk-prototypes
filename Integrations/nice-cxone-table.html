3.2.X Use Case X: NICE CXone (native CCaaS integration, P0)

Integration NICE CXone
Outcome Connect your Retell agents to NICE CXone so AI agents can receive and handle calls routed through NICE's contact center platform. Agents can be assigned to queues, handle inbound and outbound calls, and transfer to human agents with full context.
Configuration SIP trunk setup (automated or guided), NICE CXone API credentials, queue/skill assignment, transfer routing rules
Supported Actions / Tools Receive Inbound Call, Handle Outbound Dialer Call, Transfer to Human Agent (warm), Transfer to Human Agent (cold), Pass Call Context, Receive Call Metadata, Log Call Outcome, Report Agent Availability
Reference Links NICE CXone Developer Portal, NICE CXone SIP Documentation, NICE CXone API Reference

Key Requirements

Requirement Description Customer Signal Status
SIP connectivity The integration should handle SIP trunk setup between NICE CXone and Retell automatically or through a guided flow, so customers do not need to manually configure dialers, SIP URIs, or trunk credentials. This is the primary pain point today and the main reason customers need specialized hand holding from the solutions team. Morgan & Morgan (Bing Wu, Nov 2025); Mike Derlatka flagged setup complexity Not started
Call routing NICE should be able to route inbound or outbound calls to a Retell AI agent the same way it routes calls to a human agent. The Retell agent should appear as an available endpoint in NICE's routing and queue configuration. Morgan & Morgan ("directly dial in") Not started
Warm and cold transfer The integration should allow seamless handoff in both directions. A Retell AI agent should be able to transfer a caller to a human agent in NICE with full context, and a human agent in NICE should be able to transfer to a Retell AI agent. Not started
Call context and metadata When a call is routed from NICE to Retell, relevant context should come with it, such as caller ID, account number, queue name, or IVR selections. When Retell transfers back, conversation summary, intent, and collected data should be passed along so the human agent has full context. Not started
Outbound dialer integration Enterprise customers running outbound campaigns through NICE's dialer should be able to have Retell AI agents handle those calls alongside human agents through NICE's campaign management. Not started
Reporting and analytics Calls handled by Retell AI agents should be visible in NICE's reporting so enterprises have a unified view of call volume, handle time, and outcomes across both human and AI agents for capacity planning and compliance. Not started
Concurrent call scaling Enterprise contact centers may route hundreds of simultaneous calls to Retell. The integration should support the concurrency levels that NICE customers expect without requiring per call setup or manual scaling. Not started
Failure visibility If a call fails to route, drops during transfer, or the Retell agent is unavailable, customers need clear feedback in both the NICE and Retell dashboards. Silent failures in a live contact center are not acceptable. Not started
Security and compliance Enterprise customers on NICE have specific requirements around call recording, data residency, and access controls. The integration should not introduce compliance gaps or require customers to change their existing NICE security configuration. Not started
