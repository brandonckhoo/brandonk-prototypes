Flowchart illustrating how Retell AI agents check availability, book, reschedule, and cancel events on Cal.com
during live voice conversations, keeping the entire scheduling workflow inside a single call.
How it works
1
Customer requests appointment. The customer tells the Retell AI agent they want to book an appointment, schedule a call, or set up a meeting. The agent recognizes the scheduling intent.
2
Check availability. Retell calls the Cal.com API to fetch open time slots for the configured event type. Event type, timezone, and duration are set during integration configuration.
3
Cal.com returns available slots. Cal.com returns the open time slots based on the host's calendar availability, including buffer times and existing bookings.
4
Agent offers slots to caller. The agent presents a few options in natural language, for example "I have Tuesday at 2pm or Wednesday at 3pm, which works better for you?"
5
Customer picks a time. The customer confirms their preferred slot verbally.
6
Book the event. Retell calls the Cal.com API to create the booking with caller details, email, phone, and any custom fields collected during the conversation.
7
Cal.com confirms booking. Cal.com returns the booking ID and confirmation. The event is now on the host's calendar.
8
Agent confirms to caller. The agent tells the customer their appointment is booked and that a confirmation is on the way via text and email.
9
Cal.com sends SMS and email confirmation. Cal.com automatically sends both a text message and an email to the customer with the appointment details, calendar invite, meeting link, and reschedule or cancel options. Since the customer is on the phone, SMS gives them an instant confirmation they can check right away.
10
Reschedule or cancel (optional). If the customer calls back later to change or cancel, the agent can look up the booking by phone or email and reschedule or cancel directly through Cal.com, all inside the same conversation.
