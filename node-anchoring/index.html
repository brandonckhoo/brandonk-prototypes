Conductor · Feature proposal

Node Anchoring

Let Conductor attach actionable links to the exact nodes, fields, and settings that need the user's input, so they jump straight there instead of hunting the canvas. Especially for placeholder values like API keys, event IDs, and phone numbers.

The problem

Conductor tells the user what to fill in, but not where. It names cal_api_key, cal_event_type_id, and the transfer number as dead text, then leaves the user to hunt through the agent, and even a separate Integrations page, to find each one. The agent can't take a single live call until all three are set.

How it works

1

Conductor emits anchors

When Conductor builds or edits an agent, it knows the exact ID of every node, field, and connection it touched. It attaches those IDs to its message as actionable links.

2

The user clicks a target

Each placeholder becomes a "Go to node" button. Clicking pans the canvas to that node, highlights it, and opens its inspector with the field focused.

3

Conductor follows across surfaces

Targets can live in Global Settings, Node Settings, or an integration on another page. Conductor switches to the right surface and stays docked, so the user never loses context.

This generalizes — not just Cal.com

Cal.com is one worked example. Conductor can anchor to anything it builds or edits, and Retell keeps configuration in two places — so every anchor resolves to a Node Setting, a Global Setting, or a cross-surface destination.

Global Settings agent-wide
  • Default dynamic variables in the Global Prompt — {{customer_name}}, {{dob}}, {{customer_zip}}
  • Model and voice selection
  • Agent Handbook
  • Post-Call Data Extraction fields
  • Security & Fallback settings
Node Settings per node
  • Transfer numbers on Transfer & Agent Transfer nodes
  • Function parameters — event IDs, query fields, headers
  • A node's prompt or its transition conditions
  • Webhook URL on a webhook node
  • Knowledge base attached to a node
Cross-surface  Integrations connections — Cal.com, Salesforce, HubSpot, Zendesk — live on the Integrations page. Conductor navigates there, opens the connect modal, and follows you back to the agent.
