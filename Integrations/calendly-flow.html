Flowchart illustrating how Retell AI agents check availability, book, reschedule, and cancel events on Calendly
during live voice conversations, keeping the entire scheduling workflow inside a single call.
How it works
1
Customer requests appointment. The customer tells the Retell AI agent they want to book an appointment, schedule a call, or set up a meeting. The agent recognizes the scheduling intent.
2
Check availability. Retell calls the Calendly API to fetch open time slots for the selected event type. Event type, host user, timezone, and duration are set during integration configuration.
3
Calendly returns available slots. Calendly returns the open time slots based on the host's calendar availability, respecting connected calendars, buffer times, and existing bookings.
4
Agent offers slots to caller. The agent presents a few options in natural language, for example "I have Tuesday at 2pm or Wednesday at 3pm, which works better for you?"
5
Customer picks a time. The customer confirms their preferred slot verbally.
6
Book the event. Retell calls the Calendly API to create the booking with caller details, email, phone, and any required event questions collected during the conversation.
7
Calendly confirms booking. Calendly returns the booking ID and confirmation. The event is now on the host's calendar.
8
Agent confirms to caller. The agent tells the customer their appointment is booked and that a confirmation is on the way via text and email.
9
Calendly sends SMS and email confirmation. Calendly's default is email only. Retell adds SMS so the caller gets an instant text confirmation while still on the phone, with the calendar invite, meeting link, and reschedule or cancel options.
10
Reschedule or cancel (optional). If the customer calls back later to change or cancel, the agent looks up the booking by phone or email and reschedules or cancels directly through Calendly, all inside the same conversation.
