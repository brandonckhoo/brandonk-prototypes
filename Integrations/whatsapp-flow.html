Proposed architecture (strawman for engineering review).
Shows how Retell AI agents handle 2 way WhatsApp conversations using shared agent config, with optional escalation to voice
and seamless handoff to human agents within the same conversation thread.
How it works
1
Customer sends WhatsApp message. A customer initiates a conversation by messaging the business's WhatsApp number. In markets like Brazil, India, and Europe, WhatsApp is the default communication channel, so customers expect to start here instead of calling.
2
Webhook delivers message to Retell. Meta's WhatsApp Business API forwards the inbound message to Retell via a webhook. Each message carries the customer's phone number, which Retell uses to look up existing contact context.
3
Agent processes with shared config. The same agent configuration that powers voice calls (prompt, tools, connected integrations) handles the text message. No separate agent needs to be built for WhatsApp. Connected CRM, KB, and scheduling integrations work identically across channels.
4
Send reply via WhatsApp API. Retell sends the agent's response back through the Meta Business API. Response can include rich media (images, documents, quick reply buttons) to reduce friction for common options.
5
Customer receives reply. The response appears in the same WhatsApp thread. The conversation continues 2 way until the customer's need is resolved or escalation is triggered.
6
Escalation to voice call (optional). If the conversation needs voice (complex issue, confirmation flow, customer preference), the agent can initiate an outbound voice call with full chat context carried over. The customer sees a seamless transition from text to voice without repeating information.
7
Human handoff (optional). At any point, the agent can hand off to a human rep. The human takes over the same WhatsApp thread with full conversation history visible. AI can toggle back on if needed. This matches the Bajacheebo feedback: "toggle off the AI and let a human interject and respond too."
8
Human responds directly. The human rep replies via WhatsApp in the same conversation thread. From the customer's perspective, it is one continuous conversation regardless of who is responding. All messages are logged to the same conversation record for unified reporting and CRM activity.
