Conductor · Feature proposal

Node Anchoring

Conductor tells the user what to configure. It doesn't take them there. This proposal closes that gap.

The problem

Conductor names what to fill in, but not where. It writes cal_api_key, cal_event_type_id, and the transfer number as dead text, then leaves the user to hunt through the agent — and a separate Integrations page — to find each one. The agent can't take a single live call until all three are set.

The solution — Node Anchoring

Conductor knows where everything it builds lives. It turns each spot into a link. Click it and you land right there.

1

It adds a link

Every node, field, and setting Conductor changes shows up as a link in the chat.

2

You click it

The screen jumps to that spot, highlights it, and opens its settings.

3

Anywhere it lives

A node, a global setting, or an integration on another page. Conductor takes you there.

Works for anything Conductor touches, not just Cal.com.
